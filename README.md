# Lianghua 量化交易强化学习项目

## 项目概述

Lianghua 是一个基于强化学习的量化交易系统，它结合了K线图可视化和强化学习算法，旨在训练AI代理在金融市场中进行交易决策。该项目使用Python实现，主要依赖于Pygame进行图形界面展示，Stable Baselines3进行强化学习模型训练。

## 功能特性

1. **交互式K线交易模拟器**
   - 实时K线图展示
   - 成交量可视化
   - 布林带技术指标
   - 交互式交易系统（支持做多/做空）

2. **强化学习交易环境**
   - 符合Gymnasium标准的交易环境
   - 支持多种交易动作（买入、卖出、持有）
   - 基于交易盈亏的奖励机制

3. **深度强化学习训练**
   - 使用PPO算法进行模型训练
   - 支持多环境并行训练以加快收敛速度
   - 模型检查点保存和恢复功能

4. **模型评估与可视化**
   - 模型性能评估
   - 交易过程可视化
   - 可选的视频录制功能

## 核心组件

### 1. Interactive Trading Game (interactive_trading_game.py)

这是整个项目的核心可视化组件，提供了一个完整的K线图交易模拟器，包括：

- K线图和成交量展示
- 布林带技术指标绘制
- 交互式交易系统，支持手动交易
- 账户信息面板，显示资金、持仓、盈亏等信息

#### 主要功能：
- **键盘控制**：
  - `↑/↓`：缩放图表
  - `B`：买入（开多/平空）
  - `S`：卖出（开空/平多）
  - `ESC`：退出程序

- **交易机制**：
  - 支持做多（买入开多，卖出平多）
  - 支持做空（卖出开空，买入平空）
  - 实时计算浮动盈亏和总资产

### 2. Trading Environment (trading_env.py)

这是一个符合Gymnasium标准的强化学习环境，封装了交互式交易游戏，使其能够被强化学习算法使用。

#### 环境规范：
- **动作空间**：离散空间，包含3个动作
  - 0: 无操作
  - 1: 买入
  - 2: 卖出

- **观测空间**：13维连续空间
  - 价格信息（5维）：开盘价、最高价、最低价、收盘价、成交量
  - 账户信息（5维）：可用资金、持仓数量、持仓均价、浮动盈亏、总资产
  - 布林带信息（3维）：上轨、中轨、下轨

- **奖励机制**：
  - 基于总资产变化的即时奖励（每一步都会根据资产变化给予奖励）
  - 平仓时根据盈亏情况给予额外奖励或惩罚
  - 强制平仓时给予最终奖励
  - 奖励数值经过适当缩放以保证训练稳定

### 3. Training Script (train_trading.py)

该脚本使用Stable Baselines3的PPO算法对交易策略进行训练。

#### 主要特性：
- 支持并行环境训练以提高效率
- 模型检查点自动保存
- 可从中断处恢复训练
- 可配置训练参数（训练步数、保存频率等）

#### 参数说明：
- `--n-envs`: 并行环境数量（默认4）
- `--total-timesteps`: 总训练步数（默认1,000,000）
- `--save-freq`: 模型保存频率（默认50,000步）
- `--max-steps`: 每个训练回合的最大K线步数（默认5000）

### 4. Evaluation Script (eval_trading.py)

该脚本用于评估训练好的模型性能，并可将交易过程录制为视频。

#### 参数说明：
- `--out-video`: 输出视频文件路径
- `--model`: 模型文件路径
- `--max-steps`: 每个评估回合的最大K线步数（默认5000）

## 使用指南

### 1. 环境准备

首先需要安装所需的依赖包：

```bash
pip install pygame pandas numpy stable-baselines3 gymnasium
```

### 2. 数据准备

在`data`目录下放置CSV格式的历史交易数据文件，至少包含以下列：
- `datetime`: 时间戳
- `open`: 开盘价
- `high`: 最高价
- `low`: 最低价
- `close`: 收盘价
- `volume`: 成交量

### 3. 交互式交易

直接运行交互式交易游戏：

```bash
python interactive_trading_game.py
```

### 4. 模型训练

训练一个新的模型：

```bash
python train_trading.py
```

从检查点恢复训练：

```bash
修改 train_trading.py 中的 training_mode = "resume"
python train_trading.py
```

### 5. 模型评估

评估训练好的模型：

```bash
python eval_trading.py
```

## 技术细节

### 观测空间标准化

为了提高训练效果，观测空间包含了价格、账户和布林带等多种信息，有助于模型做出更全面的决策。

### 奖励设计

奖励机制是强化学习成功的关键因素之一。本项目采用以下奖励机制：
- 基于总资产变化的即时奖励（每一步都会根据资产变化给予奖励）
- 平仓时根据盈亏情况给予额外奖励或惩罚
- 强制平仓时给予最终奖励
- 奖励数值经过适当缩放以保证训练稳定

### 模型架构

使用Stable Baselines3的PPO算法，具有以下优势：
- 策略梯度方法，适合连续动作空间
- 相对稳定的训练过程
- 良好的样本效率
